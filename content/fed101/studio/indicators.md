# 经济指标数据模块

目标：建立可复用的指标库，为研报与规则模型提供**同源输入**——避免“不同图用不同口径/不同更新时点”，从而让复盘更可信。

## 指标如何选：需求驱动

我们的指标选择不是“越多越好”，而是从两类需求倒推：

1. **研报需求**：要写 NFP/CPI（以及宏观月报），需要哪些序列来支撑图表与要点
2. **规则需求**：要跑政策规则（如 Taylor），需要哪些关键输入与对照序列

数据来源以 **FRED** 为主，通过 API 获取后沉淀到本地指标库，供不同模块复用。

## 四大板块（你可以把它理解为一个最小信息集）

### 1) 非农就业（Labor / NFP）

关注点：劳动力市场是否在“再平衡”，以及压力来自需求还是供给。

常用维度：就业增长、失业率、劳动参与、工资增速等。

### 2) 通胀（Inflation）

关注点：结构与动量，而不是单一水平；尤其是“粘性 vs 可逆、扩散 vs 局部”。

常用维度：CPI/PCE（总项/核心）及关键分项结构。

### 3) 政策利率（Policy Rate）

关注点：实际政策利率路径与沟通口径，为“规则基线 vs 实际决策”提供对照。

常用维度：联邦基金目标区间/有效利率等。

### 4) 产出与增长（Output / Growth）

关注点：需求侧动能与增长压力，为“通胀风险 vs 增长压力”的权衡提供背景。

常用维度：GDP、消费/投资、工业与景气指标等。

## 这些指标如何被复用（与流程模块对齐）

- **研报模块**：从指标库取序列 → 生成图表与要点 → 形成会议材料
- **政策规则**：从指标库取关键输入 → 计算 baseline → 与实际利率对照 → 帮助解释偏离

## 口径提醒（不讲技术细节，但要讲清“为什么会错”）

- 指标的单位、季调、频率会直接影响解读，必须在研报与图表里保持一致
- 宏观数据存在修订：复盘时要知道“当时可见数据”和“最新修订数据”可能不同（后续可扩展 ALFRED 支持）
