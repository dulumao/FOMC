# 宏观事件模块

目标：建立会议窗口内的事件数据库，作为研报与讨论的现实原因来源。

- 以会议窗口为单位组织事件。
- 结构化存储主题、影响路径与时间标签。
- 支持多来源交叉验证与事件聚类。

产物：

- 月度/会议窗口事件清单
- 事件摘要与来源链接

## 模块定位与边界

该模块的任务是把新闻报道转化为可复盘的事件库。它不负责深度研究结论，也不替代研报判断，而是提供可追溯的事实链路（检索层依赖 DDGS，存在不确定性）。

## 处理流程概览

整体流程按“检索-筛选-聚合-排序-摘要”组织，输出事件清单与月度摘要。

- **时间窗口确定**：以 `month_key` 为输入生成 `start_date`/`end_date`，所有内容必须落在该范围内。  
- **两阶段检索**：  
  - 第一阶段固定 query 多轮次检索，追求高召回。覆盖主题包括：  
    - 劳工/就业冲击：罢工、工会行动、大规模裁员、招聘冻结等。  
    - 贸易/关税/制裁/出口管制：关税、贸易摩擦、制裁与实体清单等。  
    - 供应链/物流：港口拥堵、航运中断、运价冲击、运河风险等。  
    - 能源价格：油价、天然气、能源成本波动。  
    - 金融稳定：银行倒闭、流动性紧张、挤兑与信用收缩。  
    - 地缘冲突外溢：战争/升级带来的贸易与能源冲击。  
    - 公共卫生与全球冲击：疫情与公共卫生事件。  
    - 全球宏观冲击：衰退风险、滞胀、商品冲击等。  
  - 第二阶段由 LLM 从第一阶段事件中提炼关键词，再做事件级精搜，补齐定向信息。  
- **来源抓取与正文补全**：  
  - LLM 基于标题与来源域名挑选“最值得抓正文”的链接索引（偏好权威媒体与宏观相关度高的标题）。  
  - 若 LLM 未能给出可靠结果，则回退为“按顺序取前 N 条链接”抓正文。  
  - 抓正文使用 HTTP 直连拉取页面 HTML，去除 script/style/noscript 等噪音标签后抽取纯文本，并做长度截断，作为后续摘要与聚类输入。  
  - 未被抓正文的条目仍保留摘要片段，用于事件聚类与补充来源。  
  - 全部记录来源域名与链接，便于追溯。  
- **候选事件构造**：从标题/摘要/正文中提取粗粒度标签（冲击类型、传导渠道、国家/地区），形成可聚类的候选事件列表。  
- **事件聚类**：  
  - 先做轻量分桶（冲击类型/影响渠道/国家）减少规模。  
  - 冲击类型来自标题/摘要中的关键词匹配（如关税/制裁/供应链/金融稳定等）。  
  - 影响渠道由通胀/就业/增长/金融条件等词项触发，缺失则给默认主渠道。  
  - 国家/地区通过基础实体词表（如 US/China/Europe）识别，缺失则默认 US。  
  - 桶内由 LLM 做“同一事件”聚类，必要时二次合并。  
  - 不按日期聚类，允许跨日合并同一事件。  
- **评分与筛选**：基于来源数量 + 高可信来源加权形成重要度评分，重要事件优先进入最终清单，并与 LLM 二次重排结合。  
- **摘要与月报**：为每个事件生成简洁摘要，强调事实与传导渠道，并可选生成当月事件月报。

## 关键业务规则

- 严格月份过滤：无日期或日期不在窗口内的结果直接剔除。  
- 来源可信度分层：硬白名单加权，非白名单保留但降权。白名单当前为：Reuters、FT、WSJ、Bloomberg（按域名计）。  
- 多源验证优先：同一事件被多家高可信来源覆盖时重要度上调。  
- 可追溯性优先：保留来源标题、链接与摘要片段，避免仅保存结论文本。

## 输出结构

- 事件条目：标题、摘要、冲击类型、传导渠道、来源列表与重要度评分。  
- 月度摘要：按主题/渠道整理的可读叙述，附关键来源链接。

## 月度摘要（宏观月报）

月报是在事件清单之上生成的汇总文本，用于快速把握“本月宏观冲击全貌”，不是单一事件的摘要。

- 结构：首段给出全月概览；其后按通胀/就业/增长/金融稳定/供应链/地缘等主题分段。  
- 引用：每段末尾附来源链接，确保可追溯。  
- 约束：不引入事件清单之外的新信息，避免臆测。

## 事件时间线标签来源

- 日期：来自新闻条目的发布时间（解析后写入事件日期；无日期将被剔除）。  
- 事件类型：由标题/摘要中的关键词规则匹配得到（如关税/制裁/供应链/金融稳定等）。  
- 影响渠道：由通胀/就业/增长/金融条件等词项触发，缺失则赋默认主渠道。  
- Score：重要度评分，主要由来源数量 + 高可信来源加权构成，用于排序与筛选。

## LLM 在模块中的工作

- 事件关键词提炼：从第一阶段事件中抽取英文检索词，用于第二阶段精搜。  
- 正文抓取筛选：从检索结果中挑选“高价值链接”抓正文。  
- 事件聚类：在轻量分桶后，判断哪些条目属于同一事件。  
- 事件摘要：为最终事件生成简洁摘要与传导路径描述。
- 重要度重排：在规则评分基础上做二次排序与筛选（发生在摘要之后）。

## 质量控制思路

- 用严格时间窗口 + 来源权重 + 多源验证保证信噪比。  
- LLM 参与聚类与重排，但保留规则兜底，避免完全依赖模型。  
- 所有产物可回溯到原始来源，便于复盘与纠错。
