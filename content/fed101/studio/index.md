# FOMC Studio 搭建思路

本章说明 Studio 的板块定位与模块分工。主页面只保留整体框架与模块概览，具体模块在独立页面展开。

## 板块整体介绍

Studio 的核心目标是把“研究流程”工程化：数据来源统一、研报可复盘、规则可计算、会议材料可追溯。模块拆分不是为了技术分层，而是为了在同一套口径下复用产物与逻辑。

## 模块总览

- **宏观事件模块**：为会议窗口建立事件数据库，提供可追溯的事实来源与冲击脉络。
- **经济指标数据模块**：沉淀统一口径的指标库，为研报和规则模型提供同源输入。
- **NFP/CPI 研报模块**：用固定槽位写作，把数据摘要转成可对比的结论与叙事。
- **利率规则模块**：以 Taylor 系列规则作为基准线，帮助定位分歧来源。

以上模块的详细设计分别见独立页面。

## 会议材料编排（历史会议模拟）

历史会议模拟是编排层：按 meeting_id 生成并缓存完整材料链路，便于对比不同假设与版本。

## 可执行 cell 的使用原则

- 仅在需要验证结构或图表读法时使用。
- 优先用于研报图表拆解与规则参数敏感度展示。
- 避免把所有内容转为交互。
