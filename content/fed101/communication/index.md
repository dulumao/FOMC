# 沟通篇

一个不那么严肃但很实用的比喻：**货币政策像软件**。

- 决策（加息/降息/按兵不动）是内核升级。
- Statement / Minutes / 讲话，是把“内核变化”解释给世界看的 UI。

这也是为什么同样是“按兵不动”，市场反应却可能完全不同：**真正的信息往往藏在“为什么”和“接下来门槛是什么”里**。

本项目会生成一份可读的 Statement/Minutes（生成稿），并缓存到 `data/meeting_runs/<meeting_id>/`，便于复盘与对比。

## 研究问题

为什么研究里要单独把“沟通”作为一个维度？它不是写作练习，而是政策工具的一部分。

## 如果跳过，会发生什么

- 你会只盯着“动作”，错过“倾向”：市场往往对倾向更敏感。
- 你会无法解释“同样的动作为什么反应不同”（因为信息在 UI 里，不在按钮上）。

## 在反应函数中的角色

沟通材料的核心功能可以用两句话概括：

- **Statement：压缩信息、锚定预期**（短、硬、可被引用）。
- **Minutes：释放分歧、管理尾部风险**（长、密、暴露权衡）。

## 工具：去历史会议页读全文

我们刻意不在 Fed101 内做“阅读器”。原因很简单：材料生成、缓存、版本与完整上下文都在历史会议页；Fed101 更适合告诉你**该看什么、怎么看**。

如果你已经选了示例会议：用右侧「决议/纪要」快捷入口进入 `/history/<meeting_id>/decision`。
