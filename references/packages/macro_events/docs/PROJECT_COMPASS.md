# 联邦货币 LLM 委员会（暂定名）项目指南针

> 目标：用一套“工具 + 教学 + 沉浸式流程”帮助用户理解美联储如何做货币政策决策。此文档作为后续开发的对齐基准，会持续更新。

## 愿景与体验骨架
- 单一骨架，三种入口：数据 → 研报 → 规则模型 → 讨论 → 决议/纪要 → 复盘。
- 三种模式互链：
  - **FOMC101（学习模式）**：交互式说明书，展示指标意义、规则模型、提示词模板、小组件示例。
  - **历史会议模拟（流程模式）**：按会议时间线重放当期可见数据，生成研报/模型/讨论/决议，并做会后复盘。
  - **工具箱（工具模式）**：数据浏览、研报生成、规则模型计算、FedWatch 可视化、修订对比，随时可调出。
- 导航指引：
  - 主页突出“开始一场历史会议”“进入 FOMC101”，工具箱为全局入口。
  - 101 章节标注“在流程中的位置”，附“去某次会议试用”按钮；流程页右侧有“101 解释”面板，支持往返跳转。
  - 顶部 Tab 或模式切换：体验（会议流程）/ 学习（101）/ 工具（工具箱），保持状态。

## 核心模块设想
- **FOMC101**：短章节 + 可执行小组件。骨架示例：目标与框架 → 关键数据（NFP/CPI/PCE/金融条件）→ 规则模型（Taylor/Balanced/First-Difference）→ 沟通材料（Statement/Minutes 结构）→ 市场预期（FedWatch）→ 案例串读（1-2 次典型会议）。
- **历史会议模拟**（仅历史）：会议日历入口（最新历史会议标记 NEW）。步骤：①当期可见数据快照（绑定 ALFRED 版本）②自动研报③规则模型输出④多 Agent 讨论 + 投票⑤生成 Statement/Minutes 摘要⑥会后复盘（实际决议 vs 市场预期 vs 规则建议 vs 修订差异）。
  - 每步附“101 解释”卡；提供“修订对比”按钮查看当时数据 vs 最新修订。
- **工具箱**：独立页 + 流程内边栏调用，继承当前会议上下文（会议 ID、数据快照 ID）。

## 数据与版本策略
- FRED + ALFRED：存储 latest 与 vintages，记录 `series_id, vintage_date, release_date, units, seasonal_adj, last_updated`。接口支持按发布日期/窗口取“当时可见数据”，并提供 `compare_vintages(series_id, asof_date1, asof_date2)`。
- FedWatch：抓取目标利率概率分布时序，用于市场预期可视化、Agent 讨论输入、会后误差复盘。
- 会议数据快照：`meeting_id` 绑定一个数据快照 ID（as-of 版本），保证历史重放基于当期信息。

## 技术架构（建议，保持模块化）
- Monorepo，`apps/` + `packages/`：
  - `packages/data`（现 FED-TOOLS-DATA，提供抓取/清洗/版本管理与统一查询 API）
  - `packages/models`（规则模型、情景假设）
  - `packages/agents`（RAG 管线、角色/提示词模板）
  - `packages/common`（共享配置、类型、工具）
  - `apps/api`（BFF/GraphQL/REST，聚合数据和模型输出）
  - `apps/web`（前端：日历 + 流程体验 + 工具箱 + 101 章节）
- 现阶段只调整文档，代码结构暂不移动。

## 路线图（可迭代）
1) 历史会议重放 MVP：会议日历 → 数据快照 → 研报 → 规则模型 → 摘要（含 101 侧栏）。
2) 101 章节 MVP：数据篇、规则篇、沟通篇各 1 节，附“跳转到会议”按钮。
3) 工具箱 MVP：数据浏览/研报生成/规则模型/FedWatch/修订对比，支持流程内调用。
4) FedWatch 接入 + 图表：市场预期 vs 规则建议 vs 实际决议。
5) Agent 讨论：4-5 预设人格，主持人汇总与投票；使用“市场预期+模型+数据摘要”作为上下文。
6) 修订对比与复盘：ALFRED 修订对比、会后误差分析（市场定价与规则推荐差异）。
7) 进一步章节与案例：扩充 101 内容，新增经典会议案例。
- 未来会议模拟：暂缓；保留“下一次会议占位”可显示最新数据与 FedWatch，但不进入讨论/投票。

## 设计原则
- 以学习驱动：流程每步都有解释层；提示词/模板透明化，便于用户理解生成逻辑。
- 以模块化驱动：流程共用同一骨架，工具可独立调用；数据快照确保历史可复现。
- 以对比驱动：市场预期 vs 规则模型 vs 实际决议；当时数据 vs 修订数据。
